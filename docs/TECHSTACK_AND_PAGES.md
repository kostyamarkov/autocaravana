# Описание стека технологий и назначение страниц

Ниже — подробное описание используемого стека и назначение страниц/файлов проекта "инструкция для пользователя автодома". Файл поможет любому разработчику быстро включиться в работу.

**Стек технологий**

- **HTML:** статическая разметка — основа проекта, файл `index.html` служит входной точкой.
- **CSS:** `css/style.css` — стили для адаптивного интерфейса. Используется чистый CSS (без сборщиков).
- **JavaScript:** `js/app.js` — логика приложения (динамическая навигация, поиск, открытие фото). `js/data.js` — статические данные/контент (описания элементов автодома, метаданные фото, локализации).
- **Клиентские библиотеки (рекомендации):**
  - **Fuse.js** — быстрый клиентский поиск по ключевым словам (подойдёт для локального поиска без бэкенда).
  - **i18next** или простая JSON-структура локалей — для мульти-язычности (в проекте уже заложена возможность перехода на английский).
  - **PhotoSwipe** или **GLightbox** — лёгкая галерея/лайтбокс для просмотра фотографий с масштабированием и свайпами.

**Почему такой стек**

- Минимальные зависимости, проект остаётся полностью статическим и его просто развернуть (достаточно открыть `index.html`).
- Лёгкая поддержка локализации через JSON-файлы и простую замену текстов на клиенте.
- Клиентский поиск и просмотр фото не требуют серверной части — удобно для оффлайн-инструкций.

**Файлы и назначение страниц**

- `index.html` — главная (и единственная текущая) страница приложения. Должна содержать:
  - Главное меню / список элементов автодома (например: кухня, система отопления, водоснабжение, электроника, багажное отделение и т.д.).
  - Поле поиска по ключевым словам.
  - Контейнер для динамической подгрузки/отображения содержимого (детальные описания элементов).
- `css/style.css` — все стили интерфейса: сетка, карточки элементов, модальные окна/лайтбокс, адаптивность для мобильных.
- `js/data.js` — структура данных с описаниями элементов и ссылками на изображения; здесь же могут храниться структуры локалей (ru/en).
- `js/app.js` — основная логика:
  - Инициализация данных и локализации.
  - Рендер списка элементов и детальных страниц (можно через шаблонизацию или прямую вставку HTML).
  - Поиск: подключение Fuse.js или собственная фильтрация по `data.js`.
  - Открытие фото: интеграция с выбранным лайтбоксом, либо простая кастомная модальная галерея.

**Рекомендуемые страницы/вьюхи (для дальнейшей работы)**

- **Главная (список элементов)** — быстрый обзор всех компонентов автодома, краткие сниппеты и мини-иконки.
- **Детальная страница элемента** — подробное текстовое описание, характеристики, список фотографий с превью, советы по эксплуатации и безопасности.
- **Галерея / просмотр фотографии** — полноэкранный просмотр фото с возможностью листать и увеличивать.
- **Настройки / Локализация** — переключатель языка (RU/EN) и, возможно, режимы отображения (тёмная/светлая тема).

Примечание: эти вьюхи могут быть реализованы как один одностраничный интерфейс (SPA) с hash-routing (`#/element/xxx`) или как отдельные статические HTML-файлы. Для простоты и оффлайн-работы рекомендую SPA-подход в рамках `index.html`.

**Реализация ключевого функционала**

- Поиск по ключевым словам:
  - Подключить Fuse.js и индексировать `title`, `description`, `tags` из `js/data.js`.
  - При вводе — фильтровать и показывать релевантные карточки; при клике — открывать детальную вьюху.
- Просмотр фотографий:
  - ✅ **Реализовано:** Кастомная модальная галерея (lightbox) для просмотра увеличенных изображений.
  - Функциональность: клик по любому изображению (с классом `content-image` или `content-img`) открывает модальное окно с полноразмерной версией.
  - Закрытие окна: кнопка X в углу, клик вне изображения (на чёрный фон), или нажатие ESC.
  - Адаптивность: максимальный размер изображения в модали привязан к ширине текстовой части через CSS переменную `--content-max-width` (900px на десктопе, адаптивно на мобилях).
  - Реализация: HTML разметка (`#imageModal`), CSS стили в `css/style.css` (`.modal`, `.modal-content`, `.modal-close`), JavaScript логика в `js/app.js` (`openModal()`, `closeModal()`, `setupImageModal()`).
- Локализация:
  - Хранить словари в `js/data.js` или отдельных `locales/ru.json`, `locales/en.json`.
  - При переключении языка перерендеривать видимые тексты и подгружать переводы для элементов.

**Управление фотографиями и медиаконтентом**

- **Расположение фотографий:** все фотографии автодома хранятся в папке `pictures/camperPhotos/` (36 файлов: `image1.png` – `image36.png`).
- **Встроенные ссылки в контент:** фотографии интегрированы непосредственно в текстовое содержимое элементов (в `js/data.js`):
  - Каждый img-тег содержит класс `content-image` для применения CSS стилей: `<img class="content-image" src="pictures/camperPhotos/imageN.png" alt="Image N">`.
  - Все изображения **кликабельны** — при клике открывается модальное окно с увеличенной версией фото.
  - При наведении курсора на изображение: лёгкое увеличение (scale 1.02) и усиление тени для визуальной обратной связи.
  - Пары изображений, стоящих подряд без текста между ними, обёрнуты в контейнер `<div class="image-pair">...</div>` для горизонтального расположения.
  - Фотографии внедрены в нужных местах текста (рядом с соответствующими описаниями).
- **CSS стили и адаптивность:**
  - `.content-image` — применяется ко всем фотографиям; max-width: 100%, с bordeRadius и тенью для улучшенного вида.
  - `.image-pair` — flex контейнер для парных изображений, каждое изображение занимает ~50% ширины строки (flex: 1 1 calc(50% - 8px)).
  - Пары изображений автоматически переходят в вертикальный стек на мобильных устройствах (≤768px).
  - Одиночные фотографии занимают всю доступную ширину текстовой области (max-width: 100%).
- **Алгоритм использования фотографий:**
  1. При загрузке страницы `app.js` читает контент из `js/data.js`.
  2. HTML с встроенными img-тегами (с классом `content-image`) отображается на странице.
  3. CSS стили автоматически применяют макет: одиночные img — во всю ширину, парные img — рядом (50% каждая).
  4. ✅ **Реализовано:** При клике на фото открывается кастомное модальное окно (lightbox) с увеличенной версией.
  5. ✅ **Реализовано:** Пользователь может закрыть модель тремя способами: кнопка X, клик вне изображения, или клавиша ESC.
  6. Фото масштабируется адаптивно и не превышает ширину текстовой области.
- **Оптимизация:** фотографии используют формат PNG для качества. При необходимости снизить размер файлов рекомендуется:
  - Сжать PNG (инструменты: TinyPNG, ImageMagick).
  - Перейти на WebP формат (с fallback на PNG).
  - Добавить lazy-loading для изображений (атрибут `loading="lazy"`).
- **Добавление новых фотографий:** для добавления новой фотографии:
  1. Поместить файл в `pictures/camperPhotos/` с названием `imageN.png` (где N — следующий номер).
  2. В соответствующем разделе `js/data.js` вставить тег `<img class="content-image" src="pictures/camperPhotos/imageN.png" alt="Image N">`.
  3. Если новая фотография должна быть парной (рядом с другой), обернуть обе в `<div class="image-pair">...</div>`.
  4. Закоммитить и запушить изменения в Git.

**Доступность и качество UX**

- Контраст текста и размера шрифтов для чтения в полевых условиях.
- Навигация клавиатурой и семантические HTML-элементы для скринридеров.
- Адаптивная верстка (мобильная — приоритет) и оптимизация изображений для экономии трафика.

**Запуск и локальная проверка**

- Открыть `index.html` в любом браузере (двойной клик или через локальный HTTP-сервер).
- Для быстрой проверки через простой сервер (рекомендуется для корректной работы некоторых лайтбоксов):

```bash
# из корня проекта
python -m http.server 8000
# затем открыть http://localhost:8000/
```

- Подключить Fuse.js и реализовать клиентский поиск.

- Добавить простую маршрутизацию внутри `app.js` (hash-based) для прямых ссылок на элементы.
