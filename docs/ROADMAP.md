**Требуемый функционал (задача)**

**Анализ разрывов (что нужно сделать)**

- Данные: убедиться, что `js/data.js` содержит структурированные записи (id, title, description, tags, images[], locale текст).
- Поиск: подключить и настроить Fuse.js или реализовать надёжную фильтрацию/ранжирование.
- Галерея: интегрировать PhotoSwipe / GLightbox или сделать простую модалку с листанием.
- Локализация: вынести строки в `locales/ru.json` и `locales/en.json` или расширить `data.js` структурой локалей; добавить переключатель языка.
- Навигация/маршрутизация: реализовать hash-based маршруты для прямых ссылок на элементы (`#/item/<id>`).
- UX/Accessibility: адаптивная верстка, кнопки навигации, семантика.

**Этапы (Milestones)**

1) Минимальный рабочий продукт (MVP) — базовый набор (priority: Высокий)
   - Цели:
     - Структурировать `js/data.js` (примерная схема ниже).
     - Рендер списка элементов и детальной вьюхи в `index.html` через `app.js`.
     - Простейшая галерея (клик на миниатюру открывает модальное окно с навигацией).
     - Hash-маршрутизация для прямых ссылок.
   - Оценка: 1–2 дня.
   - Зависимости: нет внешних библиотек, можно сделать только на нативном JS.

2) Поиск и улучшения данных (priority: Высокий)
   - Цели:
     - Подключить Fuse.js и индексировать `title`, `description`, `tags`.
     - Добавить UI поля поиска с автокомплит/результатами "на лету".
     - Улучшить структуру данных для удобной фильтрации.
   - Оценка: 0.5–1 день.
   - Зависимости: Fuse.js (клиентская библиотека).

3) Галерея — интеграция сторонней (priority: Средний)
   - Цели:
     - Подключить PhotoSwipe или GLightbox.
     - Подготовить данные изображений (размеры/атрибуты) для корректной работы.
   - Оценка: 0.5–1 день.
   - Зависимости: выбранная библиотека + стили.

4) Локализация (priority: Средний)
   - Цели:
     - Добавить `locales/ru.json` и `locales/en.json` или структуру в `js/data.js`.
     - Реализовать переключатель языка и перерендер интерфейса.
   - Оценка: 0.5–1 день.
   - Зависимости: можно реализовать минимально без внешних библиотек; при желании подключить `i18next`.

5) Качество и доступность (priority: Низкий)
   - Цели:
     - Привести стили к хорошему контрасту, добавить keyboard navigation и aria-атрибуты.
     - Оптимизация изображений (размеры, lazy-loading).
   - Оценка: 0.5–1 день.

6) Опционально: сборка и CI (priority: Низкий)
   - Цели:
     - Настроить простую сборку/минификацию (esbuild/parcel) и проверку линтером.
     - Добавить README с инструкциями по локальному запуску.
   - Оценка: 1 день.

**Пример структуры записи в `js/data.js` (рекомендация)**

```
{
  id: "kitchen",
  title: { ru: "Кухня", en: "Kitchen" },
  description: { ru: "Описание...", en: "Description..." },
  tags: ["газ", "раковина"],
  images: ["img/kitchen-1.jpg", "img/kitchen-2.jpg"]
}
```

**Кто и что делает (роли / ownership)**

- Разработчик интерфейса: реализует рендер, маршруты, интеграции (Fuse.js, галерея).
- Контент-менеджер: заполняет `js/data.js`/`locales/*.json` содержимым и фотографиями.
- Тестировщик / QA: проверяет работу поиска, галереи, переключение языка и доступность.

**Краткий план действий на ближайшую неделю**

- День 1: Подготовить структуру данных в `js/data.js` и реализовать рендер списка/деталей (MVP).
- День 2: Подключить Fuse.js и реализовать поиск; добавить базовую фильтрацию.
- День 3: Интегрировать галерею (PhotoSwipe/GLightbox) и подготовить изображения.
- День 4: Добавить локализацию (ru/en) и переключатель языка; тестирование.
- День 5: Финальная доводка: стили, accessibility, документация (README).
